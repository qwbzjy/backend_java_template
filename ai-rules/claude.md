# AI Backend Development Rules

本项目是：

- Java Spring Boot 多模块 Maven 项目
- 电商系统后端
- 包含：
  - 用户模块
  - 商品模块
  - 订单模块
  - 支付模块
  - 物流模块

AI 必须严格遵守 ai-rules 下所有 md 文件规则。

---

# 必须遵守

开发前：

1. 先阅读 ai-rules 全部规则
2. 严格按 workflow.md 开发
3. 不允许跳步骤
4. 不允许擅自重构
5. 不允许修改数据库字段
6. 不允许删除已有代码
7. 每次仅完成一个小任务

---

# 输出要求

每次开发必须输出：

1. 当前完成内容
2. 修改文件列表
3. 下一步计划
4. 风险点

---

# 禁止行为

禁止：

- 擅自修改接口
- 擅自修改 SQL
- 擅自移动模块
- 擅自升级依赖
- 一次性生成大量代码
- 跳过 Service 直接写 Controller
- 使用伪代码
- 留 TODO 未实现

---

# 代码要求

生成代码必须：

- 可编译
- 可运行
- 符合 Spring Boot 最佳实践
- 符合 RESTful 风格
- 包含必要注释
- 包含参数校验
- 包含异常处理

---

# 开发模式

必须：

- 小步迭代
- 每次一个功能
- 先设计
- 再生成代码
- 等待确认后继续
